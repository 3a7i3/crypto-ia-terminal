# 📖 Detailed Documentation – Evolutionary Ecosystem & Dashboard (EN)

## 1. Automatic Launch

- **Full simulation**:
  ```powershell
  python orchestrate_ecosystem.py
  ```
  - Runs multi-world evolution, archives results, generates CSV/JSON/PNG.

- **Interactive dashboard (results, analysis, AutoML)**:
  ```powershell
  .\.venv\Scripts\streamlit run evolution_dashboard.py
  # or for 3D/advanced AutoML
  .\.venv\Scripts\streamlit run evolution_3d_view.py
  ```

## 2. Key Features

- **Multi-world evolution**: 4 worlds (trend, range, crash, chaos), migration, extinction, survivor export.
- **Automatic archiving**: All results (CSV, JSON, PNG, logs) are archived in `archives/` at each run.
- **Interactive dashboards**:
  - `evolution_dashboard.py`: fitness, species, survivors, images, time analysis, CSV export.
  - `evolution_3d_view.py`: 3D visualization, clustering, heatmap, scoring, AutoML, multi-world comparison, Optuna export.
- **Integrated AutoML**: Automatic parameter optimization (TP, SL, RSI, MA, etc.) with objective selection (fitness, sharpe, drawdown, return).
- **Advanced filters**: fitness, species, generation, strategy selection, CSV/JSON export.

## 3. Using the Dashboards

### a) General dashboard (`evolution_dashboard.py`)
- **World selection**: dropdown menu.
- **Fitness by generation**: interactive chart.
- **Species distribution**: dynamic area chart.
- **Survivor table**: cross-world bests.
- **Strategy parameters**: search by ID.
- **Image display**: PNGs generated by the simulation.

### b) Advanced 3D & AutoML dashboard (`evolution_3d_view.py`)
- **3D visualization**: interactive scatter plot (TP, SL, fitness, species).
- **Advanced filters**: fitness, species, generation, strategy selection.
- **KMeans clustering**: automatic segmentation of the population.
- **2D heatmap**: TP/SL density.
- **Time analysis**: species diversity over generations.
- **Multi-world comparison**: max fitness per generation for each world.
- **Automatic scoring**: top filtered strategies.
- **Advanced AutoML**: multi-parameter optimization, objective selection, Optuna results export (JSON).

## 4. Export & Archiving
- **CSV**: each population/generation, filtered export from dashboard.
- **JSON**: cross-world best survivors, Optuna results.
- **PNG**: automatic visualizations (3D, dominance, etc.).
- **Archives**: all results are copied to `archives/` at each run.

## 5. Best Practices & Tips
- **Always run the simulation before opening dashboards** (to generate required files).
- **Use filters to explore diversity** (species, fitness, generations).
- **Leverage AutoML to identify performance zones** (fine parameter optimization).
- **Export results for external analysis** (Excel, Python, etc.).
- **Check logs and archives for audit or reproducibility.**

## 6. Troubleshooting
- **No data in dashboard**: re-run `python orchestrate_ecosystem.py`.
- **Streamlit/Optuna error**: check packages are installed in venv (`pip install -r requirements.txt`).
- **Port already in use**: change Streamlit port (`--server.port 8502`).
